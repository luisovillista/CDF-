The dplyr() library

dplyr is a grammar of data manipulation, providing a consistent set of verbs that help you solve the most common data manipulation challenges:

mutate() adds new variables that are functions of existing variables select() picks variables based on their names. filter() picks cases based on their values. summarise() reduces multiple values down to a single summary. arrange() changes the ordering of the rows.

The filter verb

You use filter when you want to look/extract a subset of your observations, based on a particular condition.

Filtering for one year

Every time you apply a verb, you‚Äôll use a pipe (%>%). A pipe is a percent, greater than, percent.It says ‚Äútake whatever is before it, and feed it into the next step.‚Äù

After the pipe, we can perform our first verb. We have data on many years, but we‚Äôd like to filter for just one. Let‚Äôs say we filter for 2007, the most recent data in the dataset. Here, we‚Äôre saying we want to filter for only the observations from 2007.

Filtering two Varibles

We can specify multiple conditions in the filter. Each of the conditions is separated by a comma: here we are saying we want only the one observation for the year 2007, comma, where the country is the United States. Each of these ‚Äúequals-equals‚Äù expressions is called an argument. This kind of double filter is useful for extracting a single observation you‚Äôre interested in.

  filter(year == 2007, country == "United States")

The Arrange Verb

The arrange verb sorts the observations in a dataset, in ascending or descending order based on one of its variables. This is useful, for example, when you want to know the most extreme values in a dataset.

Sorting with Arrange Verb

The arrange verb is used after the pipe operator. You would type gapminder, then the pipe operator ‚Äú%>%‚Äù and then arrange(). Within those parentheses, you tell it what column you want to arrange by.

In the following example the observations are sorted in ascending order, with the lowest GDP per capita appearing first.

# Sorting in ascending by gdpPercap using Arrange Verb
gapminder %>% 
  arrange(gdpPercap)

The Mutate Verb

The mutate verb is used to change the values of a dataset‚Äôs variable base in other/s dataset‚Äôs variables or even the same dataset‚Äôs variable, it is also use to create a new variable within the dataset.

Changing an existing variable

You use mutate like you would filter or arrange, after a pipe operator. Inside the following mutate statement, what‚Äôs on the right of the equals sign is what‚Äôs being calculated, and what‚Äôs on the left is what‚Äôs being replaced. So, on the left, you‚Äôre saying you want to replace the existing pop column, by writing pop/million. The result is the same table, but with pop replaced with a new value, one that‚Äôs much smaller than it was before. This is how you manipulate existing variables in the table. That‚Äôs often necessary during data processing and cleaning. Just like filter and arrange, you‚Äôre not altering the original gapminder data, you‚Äôre just changing the value in this new data frame that‚Äôs being returned.

gapminder %>%
  mutate(pop = pop / 1000000)

Combining Verbs

The verbs reviewed so far, can be combine in order to obtain detail data form a dataset. Suppose we wanted to know the countries with the highest total GDP in the year 2007. We would do this in three steps- create the column, filter for 2007, then sort. We use mutate to create the total GDP column. Then we‚Äôd use filter to restrict our analysis to only the year 2007. Then we‚Äôd use arrange to sort the data in descending order of our new GDP variable. This gives us the answer we‚Äôre looking for.

gapminder %>%
  mutate(gdp = gdpPercap * pop) %>%
  filter(year == 2007) %>%
  arrange(desc(gdp))
## # A tibble: 142 √ó 7
##    country        continent  year lifeExp        pop gdpPercap     gdp

Combining filter, mutate, and arrange

Find the countries with the highest life expectancy, in months, in the year 2007.

In one sequence of pipes on the gapminder dataset:

filter() for observations from the year 2007, mutate() to create a column lifeExpMonths, calculated as 12 * lifeExp, and arrange() in descending order of that new column

Filter, mutate, and arrange the gapminder dataset

library(gapminder)
library(dplyr)

# Filter, mutate, and arrange the gapminder dataset
gapminder %>% 
    filter(year==2007) %>%
    mutate(lifeExpMonths = 12 * lifeExp) %>%
    arrange(desc(lifeExpMonths))
